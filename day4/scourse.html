Subqueries

Definition and Purpose:

Subqueries, also known as nested queries or inner queries, are queries embedded within another SQL statement. They play a crucial role in breaking down complex problems into smaller, more manageable parts. The purpose of subqueries is to retrieve data that will be used by the main query to perform additional filtering, comparisons, or calculations.

Types of Subqueries:

There are two main types of subqueries: Scalar Subqueries: These subqueries return a single value and can be used in places where an expression is expected, such as in the SELECT list or a WHERE clause.
Table Subqueries: These subqueries return a set of rows and can be used in places where a table or a set of values is expected, such as in the FROM clause or a JOIN operation.

Usage in WHERE Clause

One common use of subqueries is in the WHERE clause. Subqueries in the WHERE clause allow you to filter the result set based on the results of another query. For example, you might use a subquery to find all rows where a certain condition is met in a related table.

SQL Joins


**1. Purpose:**
SQL joins are used to combine rows from two or more tables based on a related column between them. This allows you to retrieve data from multiple tables in a single query, facilitating the extraction of meaningful information by associating related records.
**2. Types of Joins:**
There are several types of SQL joins, including:
- **INNER JOIN:** Retrieves records that have matching values in both tables.
- **LEFT (OUTER) JOIN:** Retrieves all records from the left table and the matched records from the right table. If there is no match, NULL values are returned for columns from the right table.
- **RIGHT (OUTER) JOIN:** Similar to LEFT JOIN but retrieves all records from the right table and the matched records from the left table.
- **FULL (OUTER) JOIN:** Retrieves all records when there is a match in either the left or right table. If there is no match, NULL values are returned for columns from the table with no match.
**3. Syntax:**
The basic syntax for an INNER JOIN looks like this:
```sql
SELECT column1, column2, ...
FROM table1
INNER JOIN table2 ON table1.column_name = table2.column_name; ```

SQL constraints

Data Integrity:

SQL constraints are essential for maintaining data integrity within a database. They enforce rules and restrictions on the data stored in tables, preventing the insertion of invalid or inconsistent values. Common constraints include NOT NULL to ensure a column cannot contain NULL values and UNIQUE to enforce uniqueness within a column or a combination of columns.

Primary Key Constraint:

The primary key constraint uniquely identifies each record in a table. It ensures that the values in the specified column (or columns) are unique and not NULL. A table can have only one primary key, and it serves as a reference for relationships between tables. Primary keys are often used as the basis for foreign key relationships.
